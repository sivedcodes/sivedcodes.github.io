Building Offline-First Apps with Kotlin Multiplatform: The Ustad Mobile Story

KMPKotlinOffline-FirstCompose

Ustad Mobile is an open-source Learning Management System designed for emerging markets where internet connectivity is unreliable. Built with Kotlin Multiplatform and Jetpack Compose, it serves learners across 30+ countries.

I led the Android/KMP development at Chimple Learning (Sutara Learning Foundation), where we tackled one of the hardest problems in educational technology: delivering rich learning content to students without internet.

The Challenge

  • Full offline access to courses, videos, and assessments
  • Peer-to-peer content sharing without internet infrastructure
  • Cross-platform support (Android, Web, Desktop)
  • Support for xAPI, H5P, ePub, PDF, and video content

Why Kotlin Multiplatform?

  • 50%+ code reuse between Android, Web, and Desktop
  • Native performance — no bridge overhead for Nearby API
  • Shared business logic — sync, encryption, content parsing in common code
  • Jetpack Compose — modern declarative UI with multiplatform support

Architecture

commonMain/
├── domain/       # Business logic, entities
├── data/         # Repositories, data sources
├── network/      # Ktor HTTP client, sync engine
└── platform/     # Expect/actual declarations

androidMain/      # Nearby API, SQLite, Camera
desktopMain/      # File system access
webMain/          # Browser storage, PWA

Google Nearby API Integration

The core innovation was using Google Nearby Connections API for peer-to-peer file sharing. In a classroom of 30 students, one downloads content via WiFi and others receive it via peer-to-peer sync — achieving 97%+ bandwidth reduction.

Results

MetricBeforeAfter KMP + Nearby API
Bandwidth per classroom~500 MB/day<15 MB/day

Meshrabiya: The Mesh Layer

We also built Meshrabiya, the mesh networking layer powering Ustad Mobile's sync — enabling WiFi communication without any access point using virtual IP addressing and UDP packet routing.
